Progressive Enhancement

Progressive Enhancement is a web term used for the practice of creating code with the intention of bringing it to as many people as possible. https://www.freecodecamp.org summarises PE with "it gives us basic user experience and cross compatibility across browsers to ensure stability." As time moves on, browsers begin to become decommissioned or unsupported over time. PE idealizes the aespect of ensuring stuff like basic content and web functionality should be accesible and working on all type of browsers to ensure a streamlined experience.

PE can be split up itno 3 different catagories, sort of like a peanut M&M. Theres your main course, the peanut (Initial HTML), your candy coating (External CSS) and finally your candy shell (Javascript). These all work together in PE to creat a unified web page workflow to ensure functionality across all browsers. I personally believe it is increadibly important to keep practicing PE. It's an overall easier endeavor that allows web developers an easier time to create their websites with a PE design philosophy. No need to create seperate HTML or CSS for different browsers, have a simple system thats effective across all browsers. It's good for buisness as it allows consumers across the internet to access web pages regardless of the browser, and it allows the web developer to experience a relif of stress.
